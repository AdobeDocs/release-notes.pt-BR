## Experience Cloud

### Serviço da Experience Cloud ID {#mcvid}

Esta versão aprimora o código do serviço de ID para a versão 1.7.0. Inclui recursos, correções de bugs e outras alterações descritas a seguir.

**Correções e melhorias**

* Adicionado ` idSyncAttachIframeOnWindowLoad` como um sinalizador booleano opcional, que pode ser definido na função ` Visitor.getInstance`. Quando ` idSyncAttachIframeOnWindowLoad= true`, o serviço de ID carrega o iFrame de sincronização de ID na janela. Por padrão, o serviço de ID carrega o iFrame o mais rápido possível. Esse sinalizador *substitui* ` idSyncAttachIframeASAP`, que está obsoleto. Consulte [Variáveis usadas pela função getInstance do serviço de ID](https://docs.adobe.com/content/help/en/id-service/using/home.html/mcvid-variables.html). (MCID-122)
* Adicionado o recuso compatível com o rastreamento de [!DNL  Marketing Cloud] IDs por domínios, aplicativos nativos e híbridos em transições da Web. Consulte [Função de ajuda Anexar ID do visitante](https://docs.adobe.com/content/help/en/id-service/using/home.html/mcvid-appendvisitorid.html). (MCID-42)
* Adicionadas funções ao código visitorAPI.js que especificam se o serviço de ID gerou a [!DNL  Marketing Cloud] ID de visitante ou de servidor, ou se as chamadas atingirem o tempo limite. Consulte [Funções de rastreamento de tempo limite](https://docs.adobe.com/content/help/en/id-service/using/home.html/mcvid-timeout-functions.html) e [Rastreamento da geração de ID de visitante do lado do cliente](https://docs.adobe.com/content/help/en/id-service/using/home.html/mcvid-client-side-id.html). (MCID-116)

**Problemas conhecidos**

Clientes que usarem os códigos DIL do [!DNL  Audience Manager] e visitorAPI.js na mesma página deverão definir a variável DIL como ` secureDataCollection= false`. Consulte [secureDataCollection](https://marketing.adobe.com/resources/help/en_US/aam/dil-secure-data-collection.html).

**Versão de manutenção: 18 de agosto de 2016**

* Corrigido um erro no [!UICONTROL Relatório de status de onboarding] que movia o intervalo de data para um dia anterior ao selecionado. (AAM-25484)
* Corrigido um link quebrado para a documentação de ajuda nas [!UICONTROL Configurações da fonte de dados]. (AAM-25447)

**Correções de erros do DIL, melhorias e código obsoleto**

DIL é a coleção de dados de API do [!DNL  Audience Manager]. A versão de julho aprimora o DIL para a versão 6.5. Atualize para a versão mais recente para aproveitar esses recursos e correções.

* Correção de um bug que impedia a implantação do código de DIL no Firefox. (AAM-24708)
* Adicionado um atributo de título ao iFrame DIL. O título de um iFrame ajuda os leitores de tela a fornecerem informações de página para os usuários que precisam de assistência ao interagirem com o conteúdo online. (AAM-25317)
* Adição de suporte a DIL para CORS (Cross-Origem Resource Sharing, compartilhamento de recursos entre ). Consulte [useCorsOnly](https://marketing.adobe.com/resources/help/en_US/aam/dil-use-cors-only.html). (AAM-25319, AAM-20139).
* O iFrame DIL é compatível com atributos ` iframe.sandbox='allow-scripts allow-same-origin';`. Para aumentar a segurança, permita somente esses 2 tokens. Para obter mais informações, consulte a seção Atributos na [Documentação do iFrame](https://developer.mozilla.org/pt-BR/docs/Web/HTML/Element/iframe). (AAM-23335)
* Agora, o DIL efetua chamadas HTTPS seguras por padrão. Clientes que usarem os códigos DIL do [!DNL  Audience Manager] e visitorAPI.js na mesma página deverão definir a variável DIL como ` secureDataCollection= false`. Consulte [secureDataCollection](https://marketing.adobe.com/resources/help/en_US/aam/dil-secure-data-collection.html).

**Correções, melhorias e desaprovações**

* Aumento na frequência e na vida útil da coleta de dados das métricas do Facebook. O novo período de coleta de dados é o seguinte:

   * Todas as métricas do Facebook são coletadas a cada hora durante as primeiras 72 horas após uma publicação.
   * Continue a coleta de dados a cada 8 horas nas próximas 72 horas.
   * Depois disso, continue a coleta de dados diariamente para os próximos 60 dias.

* Adição de uma coluna de Segmentação geográfica para o Twitter na exportação de relatório Análises de postagem.
* Cinco novas métricas de reação do Facebook estão disponíveis nos relatórios Análise de publicação e Análise de propriedade. Eles incluem: Amor, Uau, Triste, Haha e Raiva. Em uma versão que será lançada em breve, as métricas do Facebook também serão disponibilizadas nos relatórios do Adobe Analytics.
* Correção um problema que, ao clicar na lupa em uma postagem privada do Twitter usando o [!DNL  Social Buzz], redirecionava o usuário para a mesma página, em vez de abrir a postagem original no Twitter.
* Corrigido um problema que impedia que tweets escalados fossem reivindicados.
* Correção de um problema em que os relatórios do Social Buzz gerados em dias diferentes removiam determinadas postagens.
* Correção de um problema em que alterar as configurações de feed na Moderação unificada não era salvo após o logout e o login novamente.
* Correção de um problema em que URLs em tweets mencionados ocuparam mais de 24 caracteres.
