**Correções**

* Para pequenas correções de erros e aprimoramentos
de desempenho, consulte [Adobe Mobile Services](https://docs.adobe.com/content/help/pt-BR/mobile-services/using/home.html) para obter a documentação do produto. Para visualizar as notas de versão de versões anteriores, abra as *Notas de versões anteriores* no painel à esquerda.

**Atualização em 23 de agosto de 2017**

A partir de 22 de agosto, a Adobe Analytics começará a determinar geo_zip em todas as ocorrências recebidas durante a coleta de dados. Essa atualização melhora a precisão dos dados, particularmente para dispositivos móveis. Anteriormente, o geo_zip era definido uma vez por visita. Agora, geo_zip pode ser alterado corretamente durante a visita de um visitante. O campo geo_zip não é usado em relatórios, mas é exibido nos Feeds de dados.

**Atualização em 21 de agosto de 2017**

Em 22 de agosto, o Adobe Analytics habilitará relatórios de [Segmentação geográfica](https://marketing.adobe.com/resources/help/en_US/reference/reports_geosegmentation.html) para todos os clientes. Essa alteração antecipa a futura remoção das configurações de segmentação geográfica no Admin Console.

Data de lançamento: **17 de agosto de 2017**

>[!IMPORTANT]
>
>A partir da versão de setembro de 2017, a Ad Hoc Analysis encerrará o suporte para versões Java anteriores à 1.7.0_76. Nossos arquivos .jar são assinados com uma criptografia segura de 256 bits que não é suportada pelas versões Java anteriores à 1.7.0_76. Esse certificado de 256 bits nos permite oferecer segurança aprimorada. A versão de agosto de 2017 ainda oferecerá suporte a versões anteriores. Além disso, a partir de setembro de 2017, a página de inicialização da Ad Hoc Analysis incluirá temporariamente um link para uma versão não suportada do produto para uso com as versões Java anteriores à 1.7.0_76. Se você estiver usando o Java 1.7.0_76 ou posterior, comece a trabalhar com a sua equipe de TI para atualizar com segurança para uma versão mais recente do Java imediatamente.

**Correções, melhorias e desaprovações**

* Correção de um bug visual na visualização de destino em que um ícone extra de dica de ferramenta aparecia sempre que um usuário selecionava outro valor na janela suspensa Dispositivos. (AAM-28010)
* Os usuários vinculados às IDs [de](https://docs.adobe.com/content/help/pt-BR/audience-manager/user-guide/reference/ids-in-aam.html) organização agora podem fazer logon na interface do usuário do Audience Manager com seu Adobe ID em vez de nome de usuário e senha. (AAM-28161)
* A tela de Pesquisa no Audience Marketplace agora inclui um link para o [Localizador de público-alvo](https://www.adobe-audience-finder.com/). Use o Localizador de Audiências para descobrir parceiros que podem fornecer o tipo certo de feed de dados para sua campanha. (AAM-30241)
* Correção de um problema em que os usuários podiam excluir mapeamentos de segmento de teste no Laboratório de Audiências, mesmo que fossem usados em um destino. (AAM-29964)
* Os usuários com permissões de administrador recebem uma mensagem de email automática quando o acesso a um feed de dados compartilhado é revogado. As mensagens de email agora incluem um link no email para o site de documentação do produto, explicando por que o acesso foi revogado. (AAM-30625)
* Correção de um bug no qual a janela modal de destinos ultrapassava o tamanho da janela do navegador. (AAM-30893)
* As características da pasta são excluídas da lista de traços influentes na modelagem algorítmica. (AAM-30960)
* Correção de um erro em que a exclusão de um mapeamento de destino demorava muito tempo. (AAM-31338)
* Corrigido um erro de interface do usuário no qual as métricas relacionadas à Audiência endereçável mostravam zero em vez de N/A para métricas não aplicáveis. (AAM-31712)
* Correção de um problema em que a interface do usuário demorava a responder em várias páginas para clientes com centenas de fontes de dados configuradas. (AAM-31788)
